## Runtime Orchestration (Google ADK)

Google ADK is the runtime agent framework. All 6 agents run as `LlmAgent` instances:

```python
# The ADK pipeline (simplified from orchestrator.py)
observer_agent = LlmAgent(name="observer", model="gemini-2.5-flash", tools=[...])
analyst_agent  = LlmAgent(name="analyst",  model="gemini-2.5-pro",  tools=[...])
strategist_agent = LlmAgent(name="strategist", model="gemini-2.5-pro", tools=[...])
operator_agent = LlmAgent(name="operator", model="gemini-2.5-flash", tools=[...])
auditor_agent  = LlmAgent(name="auditor",  model="gemini-2.5-pro",  tools=[...])

# Conditional routing based on Analyst confidence
if confidence >= 0.65:
    # HIGH CONFIDENCE: dispatch first, Auditor sweeps post-dispatch
    pipeline = Observer → Analyst → Strategist → Operator → Auditor
else:
    # LOW CONFIDENCE: Auditor verifies before any dispatch
    pipeline = Observer → Analyst → Auditor → {VERIFY → Strategist → Operator}
                                             → {RETRACT → Pipeline STOP}
                                             → {INVESTIGATE → Human Escalation}
```

> **Note:** The code block above shows the **5-agent reactive pipeline** that runs on every signal. The **6th agent (Predictor)** runs independently — it analyzes weather forecasts to proactively pre-position resources before a crisis hits. See `predictor.py`.

### Decision Points Made by ADK-Orchestrated Agents

| Decision | Where | Logic |
|---|---|---|
| Cluster or new crisis? | After Observer | Geo + time + type proximity via Analyst signal fusion |
| Route to Auditor first? | After Analyst | `confidence < 0.65` → Auditor verification branch |
| Resource allocation | Strategist | mortality × PSER vulnerability × travel_time, constrained optimizer |
| Retract or proceed? | Auditor | Cross-reference rescue calls, sensors, hospital admissions |
| Staged or full alert? | Operator | Side-effect detection from traffic congestion spikes |
| Retroactive upgrade? | Analyst (fusion) | Clusters new signals with old low-confidence ones, cross-references sensors |

### ADK Benefits Demonstrated

- **Built-in tracing:** Every agent invocation emits structured trace events (tool calls, reasoning, decisions) — piped to WebSocket `/ws/trace` for live dashboard display
- **Tool registry:** All 20 integrations are registered `FunctionTool` instances visible in trace logs
- **Conditional flows:** Pipeline branches on Analyst confidence for pre-dispatch vs post-dispatch auditing
- **Memory + state:** Built-in session state passes structured data between agents
- **Direct Vertex AI path:** Production deployment is one config change away
